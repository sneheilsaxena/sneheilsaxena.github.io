"Why React is Awesome": A Response & Critique

I've used React in a variety of settings, both personal and professional, and therefore I've naturally watched a lot of tutorials and explanation videos about it. Towards the beginning of my journey with it, I came across this video:

As a beginner at not just using React but just client side web development in general, the video made me feel very excited to use React for projects! Now that I have more experience with it however, I want to go back and watch the video to see how my opinions about how it presents things has changed.

The video starts with a description of React - what it is, where it came from, what it does, and most importantly, what it doesn't do, which I really appreciate. It's all the more important to know the limitations of any new tool you're trying to pick up as a beginner, and I appreciate that Chris takes that into account. He mentions that it’s primarily a client-side JavaScript library made by developers at Facebook and that using it for server-side functionality is uncommon, though technically possible.

Chris makes an argument for why the library will be adopted even more as time goes on due to changes in its licensing no longer requiring developers to stop using it in case they’re in a lawsuit with Facebook. So far, the arguments are presented well – some time has passed since the video was uploaded and looking at the data, we can indeed see that React usage has gone up exponentially.


In fact, I have observed this anecdotally too from interviewing with different companies and noticing how many of them either already use React or are transitioning to it. Chris says that Angular’s “bi-directional” flow is harder to understand and implement compared to React’s, and in my opinion that is a bit of an apples-oranges comparison. As Mosh writes here, Angular is a full fledged framework for web development with stronger opinions about how the application should be structures as opposed to React, which in the majority of cases is just focused on designing what the user sees. Coupled with his mention of MVC architecture in the intro, I would have liked to see this point being made here as it provides more clarity for beginners, who I presume are the primary audience for this video. Slight cherrypicking, but one of the comments on the video even asks if they need to learn HTML and CSS before using React, which in layman’s terms is like asking to do acrobatics before learning how to walk.

I do find the description of how React works and how it’s a “decorative” framework accurate based on my and my past coworkers’ experience. Once again however, during the comparison with Vue and Angular I would have appreciated the framing of the narrative to not just be “React is better and easier to use than Angular for front-end development” and be presented as “React is only focused on front-end, and therefore for those looking to do just front-end React is a better choice than trying to use Angular which is more complex since it does more”.
